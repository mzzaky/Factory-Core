Overview

The Achievement System adds an engaging layer of progression implicitly connected to every fundamental module within your industrial ecosystem. From the first spark of production to controlling an immense multi-million capitalist empire, this system monitors player stats to award them with impressive customized popups, sounds, and an undeniable sense of accomplishment as they unlock distinct milestones.

üí° Highly Customizable
Everything from achievement names, icons, thresholds, and even distinct pitch variations on sound effects can be meticulously tuned within achievement.yml.

How It Works

Achievements represent significant benchmarks divided into two main categories:

  • First-time Achievements: These are granted the precise instant a player executes a milestone action for the first time. E.g., purchasing their first factory property or completing their first marketplace listing.
  • Cumulative Achievements: These require dedication. The plugin silently counts cumulative expenses and yields over an indefinite period (e.g., spending $500,000 in employee salaries). The precise moment a threshold crosses the defined target, the achievement triggers permanently.
  • Offline Progression: Your empire never sleeps. For things like marketplace sales that might occur while the factory owner is offline, their progress is intelligently added. They'll receive their proper notifications the very next time they connect.
üèÜ Toast Notifications
When a player satisfies the conditions, an immediate vibrant on-screen toast is presented. This includes a prominent ACHIEVEMENT UNLOCKED title card accompanied by the tailored sound effect, ensuring the moment feels triumphant!

Configuration & Data

The achievement definitions and global configurations natively reside in achievement.yml.

Configuration Details

Key Definition
name & description The display interface formatting natively supporting standard & color codes.
type The specific trigger enumerator hooking inherently to the plugin's background events. E.g., CUMULATIVE_PRODUCTION_COST.
threshold The numeric value designating when an achievement is officially conquered (only explicitly applicable to CUMULATIVE_* triggers).
sound Standard XSound library string enum validating the SFX deployed in the toast. E.g., UI_TOAST_CHALLENGE_COMPLETE.
toast-fade-in / stay / fade-out Configurable timing arguments defined directly in server ticks globally controlling how long the title displays persist on-screen.

Player Save Data

Actual progression metrics, cumulative floating-point values, and the permanent unlockTimestamps for when an achievement was awarded are flawlessly embedded in data/achievements.yml utilizing absolute player UUID representations.

Developer Tips

üë®‚Äçüíª Accessing the Manager API
If you are utilizing our custom mechanics or overriding them via addons, you effortlessly plug into our AchievementManager instance to manipulate player progression dynamically.

You can seamlessly inject incremental stat modifications directly into a player's achievement pool using a simple API endpoint call.

JAVA 
// Retrieve the primary manager from the core plugin instance
AchievementManager achManager = plugin.getAchievementManager();

// Example 1: Instantly awarding a FIRST-TIME achievement
achManager.awardAchievement(player, "new_investor");

// Example 2: Programmatically progressing a cumulative track
// If threshold reaches its maximum, the function handles the toast popup automatically.
double transactionAmount = 15000.0;
achManager.addProgress(player, "relentless_grinder", transactionAmount);

// Example 3: Modifying offline player accumulation dynamically
achManager.addProgressOffline(offlinePlayerUUID, "capitalist_monopoly", 500.0);
